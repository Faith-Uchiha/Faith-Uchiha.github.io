换了新电脑后一直没有更新博客，只是把文章都同步了下来。

今天想要hexo g&& hexo d的时候竟然出现

的问题。也不知道是什么回事。搜索后大致定位问题出现在ssh认证上。但是也不知道在哪有问题啊，当初新建hexo博客的时候也就按着步骤一步一步走就完了，ssh也没明白是什么回事，今天才了解。

排查过程

  1. 首先用下列命令检查是哪里出了问题。

    最后几行会看到 id_rsa、id_rsa.pub两个文件的路径名。和permission denied(publickey)的错误

  2. 去C:/用户/.ssh/目录(我的是Windows系统)，发现并没有这两个文件。于是使用下面命令生成ssh密钥对

    执行完毕后，这两个文件就生成了

  3. 本以为这样就结束了。欢天喜地的再执行ssh -T git@github.com,发现还是permission denied。？？？
    原来是我没有在github的个人settings->SSH And GPG keys中添加公钥(id_rsa.pub的内容)。
    这里就要简单的讲一下ssh的原理了：是一种安全传输方式。利用非对称加密的方式来进行身份验证。非对称加密存在公钥和私钥密钥对，他们是成对匹配的。公钥放在服务器上，别人可以获取；而私钥存放在本地，别人无法截获。当你想要连接服务器时，会把私钥发送过去，服务器会测试公钥和私钥是否匹配，匹配则可以建立连接，推送代码。

    理解了原理之后，为什么部署失败，也就清楚了。

Linux环境安装redis

发表于 2018-04-01 | 更新于 2018-04-01 16:10:25 | 分类于 部署
字数统计: 298 字 | 阅读时长 ≈ 1 分钟
  1. yum install gcc-c++。 为编译做准备
  2. 通过SSH将redis-版本.tar.gz 发送到Linux某个文件夹。Linux的Terminal下，进入文件夹，解压压缩包到指定路径 tar -xvf redis-版本.tar.gz -C /usr/local/
  3. 进入解压出来的文件夹redis-版本号，输入命令make编译 ，这个时候只是编译出来了可执行文件，redis还未安装
  4. 使用 make PREFIX=/usr/local/redis install，安装redis到/usr/local/redis目录
  5. cd .. —> cd redis ->cd bin —>./redis-server 就将redis启动了
  6. 重新打开一个Linux连接，再进入相同的目录，就会发现一个redis-cli ，是为客户端连接redis准备的

上面的启动叫做前端启动模式，下面介绍后端启动

  1. 将redis-版本号 文件夹内的redis.conf 复制到 /usr/local/redis/bin/下
  2. 修改redis.conf配置文件，daemonize yes
  3. 启动时，要指定配置文件 ./redis-server redis.conf
  4. 关闭：ps -ef | redis kill -9 进程号 （不推荐这种方法，redis持久化数据丢失）
    正确**姿势：./server-cli shutdown
  5. 开启6379端口：/sbin/iptables -I INPUT -p tcp –dport 6379 -j ACCEPT
    /etc/rc.d/init.d/iptables save 为了jedis访问

Web应用导入Linux

  1. Eclipse中，在相应的Web项目右键->Export->war file
  2. 利用SSH连接上Linux，在Linux中开启Tomcat，把War包扔到tomcat目录下的Webapps文件夹下，会自动解压 （如果没看到解压，请在SSH中刷新一下，就可看到已经解压；若仍未解压，请检查tomcat服务器是否开启）

导入Web应用所需数据库

  1. 在Navicat中，在本机数据库名右键->转储SQL文件。导出已开发好的数据库中所有的表
  2. 在Navicat中连接到远程Linux机器的数据库，新建数据库->右键->运行SQL文件，就将本地的数据库导到远端机器里了，这样web应用就可以运行。

注意：一定要在Linux虚拟机中启动tomcat服务，在SSH中双击startup.sh是没有用的。

安装JDK1.8.0_u144

  1. 官网下载jdk-8u144-linux-i586.tar.gz
  2. 使用SSH将文件发送到 Linux的 /soft/文件夹
  3. tar -xvf jdk-8u144-linux-i586.tar.gz -C /usr/local 解压
  4. vim /etc/profile 配置JAVA_HOME等环境变量
  5. source /etc/profile 生效

安装mysql

前面两相同，都是下载，并发送到Linux。从第三步开始，要先在/usr/local/下创建文件夹Mysql。

解压的时候，-C /usr/local/mysql

后面的步骤：安装–启动–加入服务

  1. 安装服务器端：rpm -ivh MySQL-server-5.6.22-1.el6.i686（换成你的版本号）.rpm

    安装客户端：rpm -ivh MySQL-client-5.6.22-1.el6.i686（换成你的版本）.rpm

  2. 启动mysql

    service mysql start

  3. 将mysql加到系统服务中并设置开机启动

    加入到系统服务：chkconfig –add mysql

    自动启动：chkconfig mysql on

    在后来就是改密码、开启远程登陆、开启3306端口

安装tomcat

很简单，没有难度

*.tar.gz的文件包发送到Linux–解压–开启8080端口 ，启动tomcat ./startup.sh 后，就可以在物理机用虚拟机的ip:8080访问了

Linux下Mysql常识汇总

  1. Mysql的默认密码在/root/.mysql_secret

  2. 错误日志在 /var/lib/mysql/。文件名称为主机名.err。出问题多看看日志

  3. 登陆命令： mysql -u root -p
    -u [用户名] -p 表示密码登录

  4. chkconfig –add mysql增加系统服务

    chkconfig mysql on 自动启动。

  5. 远程登录

    grant all privileges on *.* to ‘root’ @’%’identified by ‘root’

    flush privileges
    linux默认只开放22端口，所以一般来说远程连接mysql或tomcat都不行，要先打开端口

  6. 远程访问mysql

    开放Linux的对外访问的端口3306

    /sbin/iptables -I INPUT -p tcp –dport 3306 -j ACCEPT

    /etc/rc.d/init.d/iptables save —将修改永久保存到防火墙中
